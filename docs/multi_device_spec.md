# Multi-Device Switching & Pairing — 功能设计文档

> 本功能已从主分支移除。文档保留供未来参考。

## 概述

ESP32 HFP 麦克风固件支持在 2 个已配对设备间切换。通过新增的 Button 4（GPIO 27）控制切换和配对。

## 硬件

```
Button 4 → GPIO 27 → GND   (低电平有效，内部上拉)
```

## 按钮行为

| 操作 | 触发条件 | 行为 | LED 反馈 |
|------|---------|------|---------|
| 短按 | 按下 <5s 松开 | 切换到另一个已配对设备 | 绿灯设备指示 |
| 长按 | 按住 ≥5s（不松手即触发） | 清除当前设备配对 → 进配对模式 | 蓝灯快闪 |
| 配对中短按 | 配对模式中按下 | 取消配对模式 | 蓝灯熄灭 |

## 设备切换流程

```
Button 4 短按下
  ├─ 断开当前 HFP SLC + BLE + SCO
  ├─ 切换 active_dev 索引 (0↔1)
  ├─ 写入 NVS
  ├─ 绿灯闪烁指示设备号 (1闪=设备1, 2闪=设备2, ×3轮)
  ├─ 重启 BLE 广播
  └─ 等待新设备自动连接 HFP + BLE
```

## 配对模式

```
Button 4 长按 5s
  ├─ 删除当前设备的 NVS 配对记录
  ├─ 断开所有连接
  ├─ 蓝灯快速闪烁 (120ms 周期)
  ├─ 等待 30s
  │   ├─ HFP 连接建立 → 保存设备地址到 NVS → 绿常亮 3s → 退出
  │   └─ 30s 超时 → 蓝灯熄灭 → 退出
  └─ 配对中短按 Button 4 → 取消 → 蓝灯熄灭
```

## NVS 存储

| Key | 类型 | 说明 |
|-----|------|------|
| `dev_cnt` | u8 | 已配对设备数 (0-2) |
| `act_dev` | u8 | 当前活跃设备索引 (0 或 1) |
| `dev_0` | blob[6] | 设备 1 的 BT 地址 |
| `dev_1` | blob[6] | 设备 2 的 BT 地址 |

## LED 状态速查

| 场景 | 颜色 | 模式 |
|------|------|------|
| PTT + BLE 已连接 | 彩虹 | 循环流动 |
| PTT + BLE 未连接 | 红色 | 闪烁 3 次 |
| 配对模式 | 蓝色 | 快闪 (120ms) |
| 配对成功 | 绿色 | 常亮 3s |
| 设备指示 (设备 1) | 绿色 | 1 闪 × 3 |
| 设备指示 (设备 2) | 绿色 | 2 闪 × 3 |

## 移除原因

多设备切换在实际使用中体验不佳：
- 经典蓝牙被动模式不适合主动切换（HFP HF 通常由 AG 发起连接）
- 必须删除旧配对才能让新设备绑定，操作繁琐
- 蓝牙协议层面限制：同一时间只能维持 1 个 HFP SLC 连接
- 对于大部份单用户场景，单设备 + 手动重配对已足够
